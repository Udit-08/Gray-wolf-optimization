1. BIG PICTURE — WHAT DOES THIS PROJECT DO?
================================================================================

This is a Flask web application that predicts heart disease from patient data
using Logistic Regression. The clever part is that it does NOT blindly use all
available features. Instead, it uses a nature-inspired optimization algorithm
called Gray Wolf Optimization (GWO) to automatically figure out which features
(columns) are the most useful — and discards the rest.

There are TWO prediction modes:
  - Binary Classification   → Does the patient have heart disease? (Yes / No)
  - Multiclass Classification → How severe? (Class 0 = none, 1 = mild, 2 = severe)

The app exposes several REST API endpoints:
  POST /upload    → Upload the CSV dataset
  POST /train     → Run GWO feature selection + train the model
  POST /evaluate  → Get accuracy, F1, confusion matrix on test set
  POST /predict   → Predict for a new custom patient input

2. THE DATASET — heart_disease_uci.csv
================================================================================

Source      : UCI Heart Disease Dataset
Rows        : 920 patients
Columns     : 16

Raw columns:
  id       - Patient identifier (dropped during training)
  age      - Age in years
  sex      - Male / Female
  dataset  - Which hospital the data came from (Cleveland, Hungary, etc.)
  cp       - Chest pain type (typical angina, atypical, non-anginal, asymptomatic)
  trestbps - Resting blood pressure (mm Hg)
  chol     - Serum cholesterol (mg/dl)
  fbs      - Fasting blood sugar > 120 mg/dl (True/False)
  restecg  - Resting ECG results
  thalch   - Maximum heart rate achieved
  exang    - Exercise-induced angina (True/False)
  oldpeak  - ST depression induced by exercise
  slope    - Slope of the peak exercise ST segment
  ca       - Number of major vessels colored by fluoroscopy (0–3)
  thal     - Thalassemia type (normal, fixed defect, reversible defect)
  num      - TARGET: 0 = no disease, 1/2/3/4 = increasing severity

Target Engineering:
  Binary target    : num = 0 → class 0 (healthy), num >= 1 → class 1 (disease)
  3-class target   : num = 0 → class 0, num = 1/2 → class 1, num = 3/4 → class 2

After one-hot encoding categorical columns (sex, dataset, cp, fbs, restecg,
exang, slope, thal), the total number of features expands from ~13 to 29.

3. THE ML PIPELINE — STEP BY STEP
================================================================================

STEP 1: DATA LOADING (ml/preprocessing.py → load_dataset)
  - Read the CSV
  - Separate features (X) from targets (y_binary, y_3class)
  - Store raw feature column names

STEP 2: TRAIN/TEST SPLIT
  - 80% training, 20% testing
  - Stratified split (preserves class proportions in both sets)
  - Random state fixed for reproducibility

STEP 3: PREPROCESSING (ml/preprocessing.py → fit_preprocessor)
  - Fit on TRAINING data only (to avoid data leakage)
  - Numerical columns: StandardScaler (zero mean, unit variance)
  - Categorical columns: One-Hot Encoding
  - Result: X_train with 29 transformed features
  - X_test is then transformed using the SAME fitted preprocessor

STEP 4: GWO FEATURE SELECTION (ml/gwo.py)
  - GWO searches for the best binary mask (which of the 29 features to keep)
  - A fitness function trains a quick Logistic Regression and evaluates F1
  - GWO runs for several iterations to find the optimal feature subset
  (Full details in sections 4 and 5 below)

STEP 5: SMOTE — Only for Multiclass
  - Synthetic Minority Over-sampling Technique
  - The multiclass dataset is imbalanced (very few class 2 patients)
  - SMOTE generates synthetic samples for minority classes
  - Applied ONLY on training data, after the split

STEP 6: FINAL MODEL TRAINING (ml/model.py)
  - Binary: Logistic Regression trained on X_train with GWO-selected features
  - Multiclass: Logistic Regression with the optimal C value found by GWO

STEP 7: EVALUATION (ml/model.py → evaluate_binary / evaluate_multiclass)
  - Model is tested on the held-out X_test
  - Metrics: Accuracy, Precision, Recall, F1-Score, Confusion Matrix

4. GRAY WOLF OPTIMIZATION — WHAT IS IT?
================================================================================

Gray Wolf Optimization (GWO) is a metaheuristic optimization algorithm
published in 2014 by Mirjalili et al. It mimics the leadership hierarchy and
hunting strategy of gray wolves in nature.

------------------------------------------------------------
4a. THE BIOLOGICAL INSPIRATION
------------------------------------------------------------

Gray wolves live and hunt in packs. They have a strict social hierarchy:

  Alpha (α) — The leader. Makes the final decisions. Best wolf.
  Beta  (β) — Second in command. Assists alpha, best candidate to replace it.
  Delta (δ) — Third tier. Scouts, sentinels, elders. Obeys alpha and beta.
  Omega (ω) — The rest of the pack. They follow the top three.

During a hunt, wolves:
  1. Track and chase the prey (exploration — search the space broadly)
  2. Encircle the prey (narrowing down)
  3. Attack (exploitation — converge on the best solution)

GWO translates this into a mathematical optimization process.

------------------------------------------------------------
4b. HOW GWO WORKS — THE MATH (simplified)
------------------------------------------------------------

We have a "search space" — in our case, each wolf represents a possible
combination of features (which columns to use).

Each wolf is a vector of 29 numbers (one per feature after encoding).

STEP A — Initialize:
  Randomly place N wolves in the search space.
  Each wolf's position = a vector of continuous values in [0, 1].

STEP B — Evaluate fitness:
  Train a model using the features selected by each wolf.
  Measure F1 score (or accuracy). Higher = better.
  Rank all wolves. The top 3 become Alpha, Beta, Delta.

STEP C — Update positions (for each iteration):
  Every omega wolf updates its position based on where Alpha, Beta,
  and Delta are. The formula is:

    X_new = (X_alpha + X_beta + X_delta) / 3   (simplified)

  More precisely, each wolf moves toward each of the top 3 leaders
  with some random coefficients that control exploration vs exploitation.

  Two key parameters control this:
    a  = linearly decreases from 2 to 0 over iterations
         (starts wide exploration, ends with tight exploitation)
    A  = random coefficient derived from a (controls step size)
    C  = random coefficient (controls how much each wolf weighs
         the leaders' positions)

  As iterations progress:
    - Early iterations: wolves spread out and explore broadly (a ≈ 2)
    - Later iterations: wolves converge tightly on the best solution (a ≈ 0)

STEP D — Binarize positions:
  Since we need a binary decision (use feature or not), the continuous
  position value x is converted using a sigmoid (S-shape) function:

    probability = 1 / (1 + exp(-x))
    if random() < probability → feature selected (1)
    else → feature not selected (0)

  This gives each wolf a binary mask, e.g.: [1,0,1,1,0,0,1,0,...,1]
  Where 1 = "use this feature", 0 = "ignore this feature"

STEP E — Constraint enforcement:
  If a target number of features (e.g., 8) is specified, the wolf's
  binary mask is adjusted to select exactly that many features
  (by picking the ones with highest probabilities).

STEP F — Repeat B through E for all iterations.

STEP G — Return Alpha's position as the best feature mask found.

================================================================================
5. EXACTLY HOW GWO IS APPLIED IN THIS PROJECT
================================================================================

------------------------------------------------------------
5a. BINARY CLASSIFICATION GWO  (ml/gwo.py → gwo_binary)
------------------------------------------------------------

Search space dimension : 29  (one position value per encoded feature)
Number of wolves       : 10
Number of iterations   : 15 (when no target_n_features is specified)
                         10 (when target_n_features is given by user)

Data used for GWO:
  The training set (80% of data) is further split inside GWO:
    GWO training   : 80% of the training set
    GWO validation : 20% of the training set  ← fitness is measured here

FITNESS FUNCTION (fitness_function_binary):
  Given a binary mask (wolf position), the fitness function:
    1. Selects only the columns where mask = 1
    2. Trains a Logistic Regression on GWO-train subset
    3. Predicts on GWO-validation subset
    4. Returns F1-score (weighted) as the fitness value
  Higher F1 = better wolf = closer to "prey"

WHAT GWO OPTIMIZES:
  It finds the binary mask (which 29 features to turn on/off) that
  maximizes the F1-score on the validation split.

AFTER GWO:
  The best binary mask (best_wolf) is used to select features.
  The FINAL model is then retrained on the FULL training set
  (not just GWO-train) using only the selected features.

------------------------------------------------------------
5b. MULTICLASS CLASSIFICATION GWO  (ml/gwo.py → gwo_multiclass)
------------------------------------------------------------

Search space dimension : 29 + 1 = 30
  The extra dimension encodes the regularization parameter C
  for Logistic Regression (C controls model complexity).
  So wolves simultaneously search for:
    - Which features to use (29 binary decisions)
    - What C value gives the best model (1 continuous decision)

Number of wolves       : 10
Number of iterations   : 15 (no target) / 10 (with target)

Data used for GWO:
  SMOTE-resampled training data is used directly (no inner split).
  Cross-validation is used inside the fitness function instead.

FITNESS FUNCTION (fitness_function_multiclass):
  Given a wolf position:
    1. Decode the last dimension into a C value:
       C = 10^(position[-1] * 4 - 2)   → maps [0,1] to [0.01, 100]
    2. Build the binary feature mask from the first 29 dimensions
    3. Run cross-validation (likely 3-fold or 5-fold) on selected features
       with the given C value
    4. Return mean F1 as fitness

AFTER GWO:
  Returns both best_wolf (feature mask) AND best_c_val (optimal C).
  Final model is trained with these two values.

------------------------------------------------------------
5c. OPTIONAL USER CONTROL — target_n_features
------------------------------------------------------------

The user can specify how many features they want (minimum 7).
If specified:
  - GWO still runs the same way
  - But when binarizing each wolf's position, instead of using the
    sigmoid threshold independently for each feature, the top-N
    features by probability are selected (exactly N features per wolf)
  - This forces all solutions to have exactly N features

------------------------------------------------------------
5d. ITERATION-BY-ITERATION WALKTHROUGH
------------------------------------------------------------

Let's trace what happens across 15 iterations for binary GWO:

Iteration 1:
  - 10 wolves initialized randomly in 29-D space
  - Each wolf's 29 values are in [0,1] (random)
  - Sigmoid applied → binary masks → some features selected
  - 10 Logistic Regression models trained (one per wolf)
  - 10 F1 scores computed
  - Top 3 wolves → Alpha (best), Beta (2nd), Delta (3rd)
  - a = 2.0 (maximum exploration, wolves jump far)

Iterations 2–7 (early phase, exploration):
  - Each wolf moves toward Alpha, Beta, Delta
  - Large random coefficients → wolves spread across the space
  - a decreases gradually: 2.0 → ~1.1
  - New feature masks evaluated each iteration
  - Alpha, Beta, Delta updated if better wolves are found

Iterations 8–12 (middle phase):
  - a ≈ 1.0, balanced exploration and exploitation
  - Wolves begin to cluster around promising regions
  - Fitness scores generally improving

Iterations 13–15 (late phase, exploitation):
  - a approaches 0
  - Wolves tightly converge on the alpha's position
  - Fine-tuning the best feature subset
  - Final alpha = best feature mask found

Total model evaluations: 10 wolves × 15 iterations = 150 Logistic Regressions
  (each is a quick fit on a subset of the training data)

9. WHY GWO FOR FEATURE SELECTION? (vs simpler methods)
================================================================================

Traditional feature selection methods:
  - Filter methods (Chi-squared, correlation): rank features independently,
    ignore interactions between features.
  - Wrapper methods (forward/backward selection): too slow for 29 features.
  - Embedded methods (L1/Lasso): only works for linear relationships.

GWO advantages:
  - Evaluates feature COMBINATIONS, not individual features
    (a feature useless alone can be powerful combined with another)
  - Simultaneously optimizes both feature mask AND hyperparameter C
    (in multiclass mode) — no separate grid search needed
  - Nature-inspired: balances exploration (finding new regions) and
    exploitation (refining the best found so far) automatically
  - Fast enough: 10 wolves × 15 iterations = 150 evaluations (seconds, not hours)
  - Avoids local optima better than greedy search
